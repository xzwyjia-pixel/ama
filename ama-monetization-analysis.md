操作系统层永远比应用层更值钱。

## 2. 逐条展开

### 路径 1: Agent 应用商店 (最快变现, 3 个月)

**对标**: Apple App Store, Shopify App Store, OpenAI GPT Store

**AMA 的角色**: 商店的底层基础设施 — 上架审核、版本管理、健康监控、用户评价

```
卖家 (Agent 开发者)          平台 (AMA)              买家 (企业/个人)
      │                        │                         │
      ├─ 提交 Agent ──────────→├─ 安全扫描 ──────────→   │
      │                        ├─ 质量审核               │
      │                        ├─ 上架 + 定价            │
      │                        │                         │
      │                        │  ←──── 浏览/搜索 ───────┤
      │                        │  ←──── 一键安装 ───────┤
      │                        ├─ 健康监控 ──────────→   │
      │  ←── 收入分润 ─────────┤                         │
      │                平台抽 30%               按需付费/订阅
```

**收入模型**:
| 项目 | 单价 | 预估月收入 (100 买家) |
|------|------|----------------------|
| Agent 上架费 (一次性) | ¥99/个 | ¥9,900 (100 agents) |

| 平台分润 (按量) | 30% | ¥30,000 (假设 10 万次调用×¥0.01) |
| 推荐位广告 | ¥500-2000/月 | ¥10,000 |
| **月收入估算** | | **¥65,000** |

**现成资产**: 已有 130+ 活跃技能, 141 归档 — 这是**首批货源**, 不需要从零冷启动

**竞品分析**:
- OpenAI GPT Store: 量大但同质化严重, 全是 ChatGPT wrapper
- 差异化: AMA 是多框架 (Pi/Claude/Codex), 不只是 ChatGPT, 企业级安全审计

### 路径 2: MaaS — 企业 Agent 管理中台 (最高单价, 6 个月)

**对标**: HashiCorp (基础设施管理), Datadog (可观测性), Splunk (安全审计)

**痛点**: 企业用 AI Agent 后的真实困境 —

```
某中型互联网公司 (200 人研发):
├── 开发团队各自装了 50+ AI 编码 agent
├── 安全团队不知道谁在用哪些 agent
├── CTO 担心代码泄露给第三方 API
├── 财务不知道每月 AI 花了多少钱
├── 合规要求审计所有 AI 操作的日志
└── 结果: CTO 下令禁止所有 AI agent, 回到石器时代
```

**AMA 卖什么**:

| 模块 | 企业愿意付多少钱 | 为什么 |
|------|-----------------|--------|
| Agent 资产管理 (L0) | ¥5,000/月 | "我们有哪些 agent? 谁在用?" |
| 安全合规 (L6) | ¥20,000/月 | 代码不泄露, 审计可追溯, CISO 安心 |
| 成本管控 (L7) | ¥10,000/月 | "每月 AI 花了 ¥50 万, 哪些是浪费?" |
| 权限治理 (L6) | ¥15,000/月 | 实习生不能用生产环境 agent |
| 智能路由 (L2) | ¥8,000/月 | 自动选最优 agent, 不浪费 token |
| **企业年单** | **¥50-70 万/年** | |

**目标客户**: 200 人以上研发团队, AI agent 使用混乱的公司

**销售策略**:
1. 免费版: 单机版 AMA (开源), 管理 ≤20 个 agent
2. 专业版: ¥5,000/月, ≤100 agent + Dashboard
3. 企业版: ¥50,000/月, 无限 agent + SSO + 审计日志 + SLA

**参考案例**: HashiCorp Terraform Cloud — 管理基础设施的工具, 年收入 $5 亿+
AMA 的逻辑完全一样: 管理 AI agent 而非管理服务器

### 路径 3: AI 成本优化 SaaS (最痛点, 1 个月 MVP)

**对标**: Vantage.sh (云成本优化, 年收入 $2000 万+), CAST AI (K8s 成本优化)

**痛点**: AI 调用浪费惊人 —

```
现状 (无 AMA):
├── 同一个问题发给 3 个 agent, 重复消耗 3 倍 token
├── 简单问题用了高思考预算的 agent (2K 问题用了 16K 预算)
├── DeepSeek prefix cache 未命中率 60%+ (AMA 可优化到 <20%)
├── 归档 agent 还在消耗文件扫描资源
└── 估算浪费: 30-50% 的 API 费用

AMA 介入后:
├── 智能路由: 简单问题 → 低成本 agent (qwen2.5 本地)
├── 缓存优化: 统一 prompt 前缀, 命中率 80%+
├── 用量归因: 每笔调用记到具体的人和项目
└── 估算节省: 30-50% API 费用
```

**收入模型**:
```
免费增值 (Freemium):
├── 免费版: 展示浪费金额, 不自动优化
├── 付费版: ¥999/月, 自动路由 + 缓存优化
└── 销售话术: "你每月 AI 账单 ¥10 万, AMA 帮你省 ¥3-5 万, 收你 ¥999"

转化率预估: 看到浪费金额 → 付费 15-20%
```

**病毒传播**: 每个付费用户发给老板的报销单 — "本月 AI 费用 ¥5 万, 优化后 ¥3 万" — 就是最好的广告

### 路径 4: 白牌 Agent 平台 (最大想象空间, 12 个月)

**对标**: Shopify (白牌电商), WordPress (白牌 CMS)

**痛点**: 每个垂直行业都想有自己的 "AI 中控台", 但没能力从零开发

```
行业客户需求:
├── 律所: 合同审查 agent + 案例检索 agent + 法律文书 agent
├── 医院: 病历分析 agent + 影像诊断 agent + 用药审核 agent
├── 房地产: 房源评估 agent + 合同生成 agent + 客户匹配 agent
├── 教育: 备课 agent + 批改 agent + 学情分析 agent
└── 电商: 客服 agent + 选品 agent + 文案 agent

他们不需要 "AI 编码 agent" — 他们需要的是自己行业的管理平台
AMA 就是底层引擎, 白牌出去
```

**收入模型**:
```
授权费:
├── 基础版: ¥29,800/年 (单行业, 10 agent 上限)
├── 专业版: ¥98,000/年 (3 行业, 50 agent, 自定义路由)
├── 企业版: ¥298,000/年 (无限, 源码授权, 私有部署)
└── 实施服务: ¥2,000/人天 (agent 定制 + 培训)

目标: 一年签 20 个行业客户
年收入: 20 × ¥100,000 = ¥200 万 (保守)
```

**现有资产优势**: AMA 已支持 3 个框架 (Pi/Claude/Codex), 行业 agent 只需要在任一框架开发即可接入

## 5. 核心竞争力 (为什么不是大厂的机会)

| 不透明定价 | 成本归因到每个 agent/人/项目 |
